# VulnPatch 变更检查清单

## 提交前必须完成的检查

### 1. 代码变更范围检查

- [ ] 我的修改仅限于我的负责模块
- [ ] 我没有修改其他成员的模块
- [ ] 我没有修改公共契约文件（除非已征得全员同意）

### 2. 架构合规检查

- [ ] 运行架构守卫检查：`python governance/architecture_guard.py`
- [ ] 没有禁止的跨模块导入
- [ ] 没有在错误的位置实现功能

### 3. 契约测试

- [ ] 运行契约测试：`python -m pytest tests/contracts/ -v`
- [ ] 所有契约测试通过

### 4. 单元测试

- [ ] 运行我的模块测试：`python -m pytest tests/test_<my_module>/ -v`
- [ ] 所有单元测试通过

### 5. 集成测试

- [ ] 运行端到端测试（如果修改了 orchestrator）
- [ ] 验证 /scan 接口正常工作

### 6. 文档更新

- [ ] 如果模块行为有变化，更新了相关 `AGENTS.md`
- [ ] 如果架构有变化，更新了 `ARCHITECTURE.md`
- [ ] 添加了必要的代码注释

## 常见问题处理

### 发现需要修改公共契约

1. 在群里发起讨论
2. 说明修改原因和影响范围
3. 等待所有成员同意
4. 由 Core Orchestrator 负责人统一修改
5. 全员同步更新

### 发现需要调用其他模块的功能

1. 检查是否已有公共接口
2. 如果没有，向 Core Orchestrator 提出接口需求
3. 等待接口定义完成
4. 按接口实现

### 发现其他模块的 Bug

1. 不要直接修改他人模块
2. 创建 Issue 描述问题
3. @相关负责人
4. 等待负责人修复

## 紧急修复流程

如果是紧急 Bug 修复，可以：

1. 先修复并提交
2. 在 PR 中标记 "紧急修复"
3. 立即通知相关成员
4. 事后补充测试

**注意**: 紧急修复不适用于功能开发，仅用于修复严重 Bug。
